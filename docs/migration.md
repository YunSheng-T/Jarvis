# Migration guide

## Between machines (Mac ↔ Ubuntu ↔ future Fedora/Arch)

Everything user-visible lives in git. Everything machine-specific lives in either:
- `.env` (secrets — not committed)
- `~/.config/systemd/user/jarvis.service` (installed from `systemd/` template)
- downloaded model files under `models/` (regenerated by install script)

Migration reduces to:

```bash
git clone https://github.com/YunSheng-T/Jarvis.git ~/Jarvis
cd ~/Jarvis
./scripts/install-linux.sh      # or install-macos.sh
cp .env.example .env && $EDITOR .env
```

## Bringing Codex context along

- Codex Desktop threads are cloud-synced to your OpenAI account; log in on the new
  machine and history is there.
- `AGENTS.md` at the repo root is the source of truth for any coding agent —
  Codex CLI reads it automatically on `codex` in this directory.
- Optional: bring over `~/.codex/` if you have custom MCP servers / skills / plugins:
  ```bash
  # on old machine
  tar czf codex-home.tgz -C ~ .codex
  # on new machine
  tar xzf codex-home.tgz -C ~
  ```
  Then audit for absolute paths (`/Users/...`) and rewrite them to `~` / Linux equivalents.

## Ubuntu → Fedora / Arch (later)

Because system-package installs are isolated in `scripts/install-linux.sh`,
port that one file to `dnf`/`pacman` when the time comes. Nothing in `src/`
should need to change.
